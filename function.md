# 项目功能总结 (function.md)

## 1. 项目概述
本项目是一个基于前后端分离架构的 **个人 AI 实践记录与资料聚合平台**。它提供了一个全景式的 AI 技术学习与实践路线图，支持资源聚合、AI 热点跟踪、资料上传共享，并深度接入了 Gemini API 及 MiniMax API 来提供智能文本摘要与分析功能。

## 2. 技术框架
- **前端系统** (`frontend`): React 19 + TypeScript + Vite + React Router + Lucide-React，支持模块化 UI 渲染与响应式页面设计。
- **后端系统** (`backend`): Node.js + Express + TypeScript/JavaScript，依靠本地 `data.json` 提供轻量级的数据持久化存储，无需配置独立的数据库。
- **AI 接口集成**: Gemini API (提供核心深度分析与资源摘要), MiniMax API (提供长文本材料的降维分析)。
- **运维与进程管理**: 项目根目录提供 `monitor.sh` 脚本用于跨平台进程自愈、端口监听监控和定时任务调度。

## 3. 核心功能模块

### 3.1 前端应用门户
前端包含一个极具科技感、支持单页路由交互的界面，核心涵盖：
* **动态结构化路由**: 涵盖九大能力版块（AI 热点新闻、AI 实践框架、AI Agent、AI Skill、知识库、学习资料、工具库、APP 开发、网页开发）。
* **全景导航系统**: 侧边栏支持多级结构与路由联动，可在不同版块之间生成定制化的子级导航数据。
* **AI 热点与文章流 (NewsPage)**: 单页支持瀑布流式的每日热点展示，内置一键呼出“AI 深度摘要”（结合 Gemini 能力），并支持源链接跳转。
* **云端资料管理与定向上传 (FileUpload)**: 提供快捷的多文件/纯文件夹上传功能，允许按版块动态拆分隔离进行管理；自动提取文档文本内容做基础分析准备。
* **安全后台控制面板 (AdminConfigPage)**: 系统包含内置管理员授权身份（Admin / Editor 表驱动），用于控制应用侧边导航配置信息的增编删、更新密码。

### 3.2 后端服务端核心
后端为一个独立的高可用 Express 实例 (默认端口 `3001`)：
* **轻量级状态存储**: 以防篡改和结构标准化的自动巡检逻辑为主轴，利用 `data.json` 作为 NoSQL 文档模型承截所有的用户新闻、材料、技能分类以及管理员状态。
* **内置用户鉴权体系 (Auth)**: 基于 SHA-256 哈希技术校验身份，支持 Token-Based Session 维持（默认 12 小时安全期限），拦截非法越权的数据修改。
* **资料存储与处理 API (Materials API)**:
  * 构建统一接纳 Base64 及 Plain Text 内容的文档上传归档逻辑。
  * 提供 `GET /api/materials/detail/:id` 接口支持详情查询逻辑。
* **智能分析引擎 API**:
  * 提供 `/api/materials/analyze/:id` 接口调用 **MiniMax** 闭源模型输出系统学习长文的降维短文。
  * 提供独立 `gemini.ts` 集成服务承接外部结构性片段内容的清洗重组。
* **自动聚合脚本 (fetch_news)**:
  * `fetch_news.js / fetch_news.ts` 提供结构死链修复与新闻信息对齐处理能力。

### 3.3 脚本部署端与进程监管 (monitor.sh)
项目封装了自动化的 Bash DevOps 能力：
* **存活探针机制**: 无间断通过对后端 `3001` 及前端 `3000` 端口以 `curl` 心跳尝试，一旦宕机遇错将自动利用 `pkill` 及 `npm restart` 自愈服务。
* **系统化 Cronjob**: 依靠 OS 时间判定，保证每天清晨 (06:00) 定时触发自动爬升的 AI News 获取队列。

## 4. 架构特点与优势总结
* **轻便灵活的部署设计**：免去了复杂数据库依赖，直接利用精简后端文件型架构就能在弱环境下零卡点直接 `npm start` 落地起来。
* **高度耦合大模型能力**：全面打通了 “文章上传/抓取 -> 高效提取分析 -> 可视化渲染展示” 的 AI 辅助全生命周期流程。
* **组件化可扩展性强**：基于严格的数据接口约定及精梳后的 React TSX 组件群，业务上极易平滑新增不同种类的应用工具及聚合分类。

## 5. 项目设计缺失评估与规划增补特性 (依设计文档补全)
对照标准《个人门户设计文档》，现有工程逻辑侧重于 AI 集群的集成，为了匹配个人全能门户的标准，将在后续持续增补并落地以下五类核心缺失功能：

### 5.1 更立体的内容编辑容器体系
* **现阶段**：以提取静态内容（Markdown / 附件图等）及外部系统拉取为主的“文件/新闻源”载体。
* **待补充 (多模内容)**：需要补全 Tiptap 核心生态。即支持文章笔记（富文本交互式编辑）、外部优质链接卡片收藏（自动化 Meta 抓取）、以及结构化的计划待办 (Todo 进度闭环)，让门户兼具笔记和 PTD 的管理功能。

### 5.2 外观定制化与动态双系主题模型
* **现阶段**：采取高度统筹定制科技感框架方案，外观不可直接编排。
* **待补充 (UI Engine)**：支持全站层面的深浅模式无级切换（配合浏览器及操作系统级别 `prefers-color-scheme` 跟踪），并开放对 Accent Color（全局品牌强调色配置、HEX 色盘切换）及站点主结构（如侧边栏可调宽度缩放范围120-240px、定制门户 Title 注入）的功能补齐面板。

### 5.3 严格的高级路由及分类管理 
* **现阶段**：一、二级主路由由前端 Config 固定与 `data.json` 粗颗粒度生成。
* **待补充 (层级映射架构)**：补充更结构化的可完全后台拖拽重序的 `一级总览 -> 二级类级 -> 独立 Content Details` 的关联机制，确保不仅能看“版块”，也能更轻松筛选和浏览层级子域下的具体归档成果。

### 5.4 工业级管理员防御及鉴权补充 
* **现阶段**：提供 Hash 及随机字符 Session Token 验证来保护核心数据写入渠道，基础形态验证能力。
* **待补充 (防线升级)**：针对恶意爆破引入 IP 会话级速率限制 (`Rate Limit`)（比如 5 次输入错误锁 10 分钟）；将认证机制底座重构成无状态分布友好的真正工业级 JWT `Bearer` 双鉴机制 + Bcrypt (Cost=12) 加解密存储，确保持久安全运营。

### 5.5 架构下沉沉淀（无状态数据库及标准化部署）
* **待补充 (稳定化重构)**：在未来的终极形态演进中，利用 Nginx (Proxy) + PM2 搭配 Let's Encrypt （自动确保证书证书轮转的 HTTPS 技术）替代本地 Shell 脚本维护高阶稳定性；
* **底座升级**：将扁平文件的 `data.json` 改写向 `SQLite (better-sqlite3)` 单节点多并发引擎，解决规模扩大带来的 JSON IO 瓶颈，并实现按域 (`menus`、`contents`、`settings`、`admin`) 拆表独立维护管理。
